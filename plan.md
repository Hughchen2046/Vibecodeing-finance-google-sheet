# 遊戲化上癮式記帳系統專案計畫書（plan.md）

## 1. 專案定位

### 1.1 專案名稱
- 德卜希記帳手札

### 1.2 核心定位
本專案是一套**遊戲化、成就感導向、帶有儀式感回饋的記帳系統**。  
使用者透過每日記帳、持續連續打卡、完成任務、達成儲蓄目標，逐步升級角色、累積成就、解鎖獎勵，讓「記帳」從壓力源轉變為「可持續、可上癮、可視覺化成長」的財務養成體驗。

### 1.3 核心價值
- 降低記帳阻力
- 強化每日打開動機
- 培養穩定財務紀律
- 將儲蓄目標轉化為可感知的成長進度
- 透過獎勵與心理回饋提升留存率

### 1.4 一句話定義
> 一個結合記帳、角色養成、任務成就、儲蓄目標與翻牌獎勵機制的財務行為養成遊戲系統。

---

## 2. 產品願景與設計原則

### 2.1 產品願景
讓使用者每天透過簡單記帳，感覺自己不是在記流水帳，而是在推進一場「財務成長冒險」。

### 2.2 設計原則
1. **先順手，再華麗**：記帳輸入流程必須極快。
2. **先回饋，再分析**：先給使用者成就感，再給數據。
3. **可愛但不幼稚**：風格療癒、有陪伴感，但仍有成熟財務工具的可信度。
4. **成長比打卡更重要**：鼓勵好的財務行為，而不是只鼓勵多輸入資料。
5. **短期有驚喜，長期有累積**：每日小獎勵、每週進展、每月成就都要成立。
6. **Google Sheets API 友善**：資料結構與流程需適合 MVP 快速落地。

## 3. 目標使用者

### 3.1 主要族群
- 想開始記帳但很難持續的人
- 對存錢有目標、但缺乏持續動力的人
- 喜歡療癒感、儀式感、成就回饋的人
- 對目標管理、個人成長、自我追蹤有興趣的人

### 3.2 使用者痛點
- 記帳太無聊，很快放棄
- 只看到數字，感受不到進步
- 儲蓄目標太抽象，缺乏推進感
- 失手中斷後容易自暴自棄
- 多數記帳工具偏工具性，缺少情緒誘因

## 5. 功能模組規劃

### 5.1 快速記帳模組
#### 目標
讓使用者在 3 秒內完成一筆紀錄。

#### 內容
- 金額輸入優先
- 類別快速選取
- 日期預設今天
- 備註可選填
- 常用模板快速帶入
- 成功後立即觸發回饋

#### 回饋
- 經驗值 +X
- 今日任務進度更新
- 連續記帳加 1
- 翻牌機會解鎖
- 若達成條件則跳成就提示

---

### 5.2 連續記帳（Streak）模組
#### 目標
建立每日打開動機，形成穩定習慣。

#### 設計重點
- 顯示目前連續記帳天數
- 顯示歷史最高連續紀錄
- 3 / 7 / 14 / 30 天設定不同里程碑獎勵
- 中斷時避免重創心態

#### 建議機制
- 連續記帳中斷不必極端歸零，可設計：
  - 保留歷史最高紀錄
  - 提供「補簽卡」或「守護符」機制
  - 每月限定 1 次緩衝

#### 視覺表現
- 火焰條
- 星光連線
- 連續打卡曆
- 能量球累積

---

### 5.3 角色升級與經驗值模組
#### 目標
讓使用者感覺自己在變強，而不是只在輸入資料。

#### 經驗值來源
- 每日首次記帳
- 完成每日任務
- 完成每週／每月任務
- 儲蓄目標達成里程碑
- 解鎖成就
- 穩定控制某類支出

#### 等級階段範例
- Lv.1 理財初心者
- Lv.2 穩定記錄者
- Lv.3 預算守門員
- Lv.4 儲蓄實踐者
- Lv.5 現金流掌控者
- Lv.6 財務修行者
- Lv.7 目標達成王
- Lv.8 自律財務冒險家

#### 升級回饋
- 新稱號
- 新頭像框
- 新主題色
- 新卡背樣式
- 新成就展示外框
- 儀式化升級動畫

---

### 5.4 任務系統
#### 任務分類
##### 每日任務
- 完成 1 筆記帳
- 為 1 筆記錄填入分類
- 今日總支出標記完成
- 撰寫 1 則備註

##### 每週任務
- 一週完成 5 天記帳
- 本週餐飲支出低於目標
- 檢查本週儲蓄進度
- 完成一週回顧

##### 每月任務
- 月度記帳完成率達標
- 達成月儲蓄目標
- 單一支出類別低於預算
- 完成月報總結

#### 任務獎勵
- 經驗值
- 翻牌機會
- 稀有卡提升機率
- 稱號碎片 / 成就碎片
- 儲蓄增益 Buff（表現型，不直接影響真實金額）

---

### 5.5 成就系統
#### 類型
##### 行為型成就
- 第一次記帳
- 連續記帳 3 / 7 / 30 天
- 單日完成 3 筆記帳

##### 管理型成就
- 本週餐飲未超支
- 本月娛樂支出下降 20%
- 本月預算達標

##### 里程碑成就
- 累積儲蓄破 5,000 / 10,000 / 50,000
- 連續三個月有結餘
- 達成第一個大型目標

#### 成就呈現
- 成就牆
- 勳章卡片
- 稱號列表
- 展示櫃 / 收藏冊

---

### 5.6 儲蓄目標系統
#### 目標
讓抽象的「想存錢」變成具體、可視覺化、可推進的旅程。

#### 支援內容
- 自訂目標名稱
- 目標金額
- 截止日期
- 分段里程碑
- 自動計算每週 / 每日應達成值
- 顯示目前達成率

#### 視覺化方式
- 進度條
- 成長樹
- 儲蓄塔
- 冒險地圖前進格數

#### 達成回饋
- 大型成就
- 升級經驗值
- 特殊翻牌機會
- 限定稱號 / 卡背 / 角色外觀

---

### 5.7 翻牌／抽卡系統（塔羅 / 奧修禪卡風格）
#### 核心定位
這不是宗教或占卜功能，而是**帶有神秘儀式感的心理回饋機制**。

#### 觸發條件
- 完成每日首次記帳
- 達成任務時加發額外抽卡
- 達成里程碑時開啟高階翻牌

#### 卡片內容
- 卡名（例如：節制、豐盛、覺察、流動、穩定、重整）
- 卡片主題句
- 一段簡短正向訊息
- 一段財務提醒語
- 稀有度（普通 / 稀有 / 傳說）

#### 卡片方向
- 溫柔提醒
- 正向引導
- 自我覺察
- 財務節制與平衡
- 避免宗教宣稱與迷信導向

#### 收集玩法
- 卡冊收藏
- 卡片圖鑑完成率
- 特定系列蒐集成就
- 限定節日卡牌

#### 可選進階機制
- 某些卡片提供當日主題 Buff（例如文字提示或任務額外獎勵）
- 卡片和角色個性 / 主題風格連動
- 卡片可作為月度回顧中的心理標記

---

### 5.8 儀表板與進度視覺化
#### 首頁應呈現
- 今日問候
- 角色等級 / 頭像 / 稱號
- 今日是否已記帳
- 連續記帳天數
- 今日任務卡
- 本月儲蓄進度
- 最近 3 筆記錄
- 今日翻牌入口

#### 分析頁應呈現
- 本月總支出
- 類別占比
- 與上月比較
- 預算使用進度
- 目標差距
- 高風險支出提醒

## 6. 外型設計架構（Visual / UX Architecture）

### 6.1 整體風格定位
風格主軸：**溫暖療癒 + 日系簡約 + 卡片式設計 + 輕遊戲感**

#### 關鍵視覺語言
- 圓角卡片
- 柔和陰影
- 低壓迫感留白
- 微動態回饋
- 儀式感強的翻牌畫面
- 可愛但克制的角色陪伴元素

### 6.2 風格目標
- 看起來舒服，不像財務報表
- 有情緒價值，但不失秩序
- 讓使用者願意天天打開
- 手機優先，操作節奏流暢

### 6.3 色彩系統建議
#### 主色
- 奶油白
- 淺杏色
- 蜜桃粉
- 柔和橘
- 霧藍
- 淺森林綠

#### 背景角色
- 背景參考backgrounddemo.png
- 角色參考hero.png

#### 狀態色
- 安全：柔和綠
- 接近超支：暖黃

- 稀有翻牌：靛紫 / 星砂金

- 內文：簡潔、清楚、容易掃視

### 6.5 視覺元件架構

- 進度條

## 9. 上癮循環設計（Hook Model）

### 9.1 Trigger（觸發）
- 每日提醒
- 任務未完成提示
- 目標進度提醒
- 想知道今天會抽到什麼卡

### 9.2 Action（行動）
- 新增一筆記帳
- 完成日任務
- 打開翻牌頁
- 查看進度與獎勵

### 9.3 Variable Reward（變動回饋）
- 每日翻牌結果不固定
- 任務完成有不同獎勵組合
- 稀有卡出現機率不固定
- 連續記帳里程碑獎勵逐漸升級

### 9.4 Investment（投入）
- 累積個人成就
- 建立角色形象
- 收藏卡冊
- 推進儲蓄目標
- 不想讓 streak 中斷

## 11. Agent 團隊規劃（7 人配置）

本專案建議採用 **7 人 Agent Team**。  
每位 Agent 都有清楚的產出物與責任邊界，避免大家像一群很有理想但方向不同的冒險者。

### Agent 1：產品經理 PM / 系統統籌 Agent
#### 角色定位
整體規劃、優先級排序、功能定義、時程控管。

#### 核心任務
- 定義 MVP 與版本範圍
- 管理需求優先順序
- 整合各 Agent 產出
- 建立功能地圖與流程圖
- 維持產品願景一致性

#### 主要交付物
- PRD / 功能規格
- Roadmap
- 版本切分
- 驗收標準

---

### Agent 2：遊戲機制策劃 Agent
#### 角色定位
負責讓系統「有上癮循環」，而不是只有 UI 可愛。

#### 核心任務
- 設計經驗值與等級曲線
- 設計 streak 規則
- 規劃每日／每週／每月任務
- 設計成就與里程碑系統
- 設計獎勵循環與稀有度

#### 主要交付物
- 遊戲化系統規格表
- 任務清單
- 成就清單
- 等級與 EXP 規則
- 獎勵機率表

---

### Agent 3：文案架構師 / 敘事體驗 Agent
#### 角色定位
負責系統語言、卡牌文案、角色語氣、成就命名、任務敘事。

## 15. 最終專案結論

這不是單純的記帳工具，而是一套以**財務成長、成就回饋、儀式感與長期留存**為核心的遊戲化系統。  
專案的成功關鍵，不在於功能塞得多，而在於是否讓使用者每天都願意打開，並在一次次小回饋中，真的感受到自己正在變得更穩、更有掌控感、更接近儲蓄目標。
